Open XML Format SDK

The 2007 Microsoft® Office system introduces a new file format that is based on XML called Open XML Formats. Microsoft Office Word 2007, Microsoft Office Excel® 2007, and Microsoft Office PowerPoint® 2007 all use these formats as the default file format. Open XML formats are useful for developers because they are an open standard and are based on well-known technologies: ZIP and XML. Microsoft provides a library for accessing these files as part of the WinFX technologies in the System.IO.Packaging namespace. This SDK is built on top of the System.IO.Packaging API and provides strongly typed part classes to manipulate Open XML documents.

1.     System Requirement

The system requirements for the SDK are:

  • Supported Operating Systems: Windows Server 2008 ; Windows Server 2003 Service Pack 1; Windows Vista; Windows XP Service Pack 2
  • .NET Framework 3.0
  • Up to 3MB of available disk space

2.     Installation Instructions

To install the SDK:

・         Download and save setup.exe to your local machine and run it to start the setup wizard; or click ‘run’ on the download dialog to initiate the set-up wizard directly

・         Click through the wizard pages to install the SDK

o   If .NET Framework 3.0 is not already installed on the machine, the set-up wizard asks you to install .NET Framework 3.0 first

o   The default path of the SDK is %systemdrive%\Program Files\OpenXMLSDK\[SDK Version No]\. You can change the path

・         If there is no error or issue, click ‘Finish’ on the last page to complete the installation

If an earlier version of the SDK (June2007 CTP, April2008 CTP) was installed, set up will first uninstall that version and do a clean installation of the SDK.

3.     Uninstalling the SDK

To uninstall the SDK, go to Control Panel and click Add or Remove Programs.

4.     Content in the SDK

The SDK contains the API (lib\DocumentFormat.OpenXml.dll), documentation (doc\OpenXMLSDK.chm), license IntelliSense xml file (DocumentFormat.OpenXML.xml) and this readme.htm file.

The OpenXMLSDK.chm file includes useful how-to topics and an extensive class library reference.

5.     What is New in this SDK

5.1 New overload ValidateXml() function

This release adds an Overload ValidateXml() function to take an XmlSchemaSet object. User is able to validate the part with a set of schema rather than one schema file.

public void ValidateXml(XmlSchemaSet schemas, ValidationgEventHandler validationEventHandler)

5.2 New Part

Mail Merge Recipient Data Part

Content Type

application/vnd.openxmlformats-officedocument.wordprocessingml.mailMergeRecipientData+xml

Root Namespace

http://schemas.openxmlformats.org/wordprocessingml/2006/main

Source Relationship

http://schemas.openxmlformats.org/officeDocument/2006/relationships/ recipientData

Extension: xml

An instance of this part type contains the document’s mail merge recipient data.

A package shall contain no more than one Microsoft namespace specific mail merge recipient data parts. If it exists, it shall be the target of an explicit relationship from the Document Settings part.

Root element: recipients

A mail Merge recipient part shall be located within the package containing the relationships part (expressed syntactically, the TargetMode attribute of the Relationship element shall be Internal).

A mail Merge recipient part shall not have implicit or explicit relationships to any other part defined by this Standard.

 

5.3 Content Type fix-up

There were a few OOXML specification bugs in describing the part content types. This release fix a few of them:

a.       WordprocessingML

                                                               i.      Alternative Format Import Part

Content Type:

One of the following formats:

Text = application/text/plain

HTML = application/text/html

WordprocessingML =
application/vnd.openxmlformats-officedocument.wordprocessingml.document

XHTML = application/xhtml+xml

 

Remove: RTF, XML, Add WordprocessingML and XHTML

Root Namespace:

not applicable

Source Relationship:

http://schemas.openxmlformats.org/officeDocument/2006/relationships/aFChunk

 

b.      SpreadsheetML

                                                               i.      Comments part is no longer permitted as the relationship target of Chartsheet Part.

                                                             ii.      Content type for Custom Property Part is relaxed to permit any content:

Content Type:

Any content, support for which is application-defined.

[Note: Some examples of formats which might be supported include:

application/vnd.openxmlformats-officedocument.spreadsheetml.customProperty

application/xml

end note]

Root Namespace:

Not applicable

Source Relationship:

http://schemas.openxmlformats.org/officeDocument/2006/relationships/customProperty

 

                                                            iii.      Pivot Table Cache Records Part’s source relationship is changed from http://schemas.openxmlformats.org/officeDocument/2006/relationships/spreadsheetml/pivotCacheRecords to http://schemas.openxmlformats.org/officeDocument/2006/relationships/pivotCacheRecords

                                                           iv.      Pivot Table Cache Record Part’s validation is changed to allow zero or one PivotTable Cache Record Part per pivot table.

c.       Presentation ML             

                                                               i.      Presentation Part: it is no longer permitted to have implicit relationships to Comments Part ; it is permitted now to have implicit relationships to Comment Authors Part; it is no longer permitted to have explicit relationships to Slide Layout Part.

                                                             ii.      Slide Layout Part: validation constrain has changed to be the target of an explicit relationship of Slide Master part and implicit relationship from Slide Part.

                                                            iii.      Slide Master Part: the relationship from Slide Master Part to Slide Layout part is changed from implicit to explicit.

d.      Shared ML

                                                               i.      Audio Part: listed supported audio types have added 2 more (audio/ogg, audio/mpeg) and removed 1 (audio/x-ms-wax) from the example.

                                                             ii.      Video Part: listed supported audio types have added 2 more (video/ogg, video/vc1) and removed 2 (video/x-ms-asf, video/x-ms-wm) from the example.

5.4 Branding related changes:

1.     Known Issues

6.1 The new blank Word file that uses the API cannot be opened with Microsoft Office Word 2007.

In this SDK, API only manipulates the parts in the document. In other words, the user has to set the right content type for the document. Please see the how-to topic “How to: Create an Office Open XML Package by using the Open XML Object Model” to see how to create a blank word document that can be opened by Microsoft Office Word.

 

6.2 After deleting all slide parts from a presentation document, reopening the file in PowerPoint still shows empty pages.

This is the recovery behavior for Power Point. The parts are still deleted from the package.

6.3 Cannot open a password protected package.

The error message is not relevant.

6.4 Excel binary file cannot be opened using this API.

Binary files are not supported in this SDK.

6.5 Theme Package cannot be opened using this API.

Theme packages are not supported in this SDK.

6.6 In the circular reference graph described below, the code snippet to remove a part from all its parent parts throws an exception.

The following code throws an exception:

 

void ClearPart(OpenXmlPart part)

{

            List<OpenXmlPart> parentParts = new List<OpenXmlPart>();

            foreach (OpenXmlPart parent in part.GetParentParts( ) )

            {

                parentParts.Add(parent);

            }

 

            foreach (OpenXmlPart parent in parentParts)

            {

                parent.DeletePart(part);

            }

}

DeletePart() removes parts recursively and disposes of all orphaned parts; any further operation on the disposed parts throws an exception. The alternative solution is to check the parent parts before calling DeletePart(). We will address this issue in a better way in future releases.

6.7 When  a Chart is removed from a document, the document cannot be opened again.

The operation to remove chart parts does not touch the references to the chart in the main document.xml. These references to the non-existent parts makes the document invalid to Word.

6.8 API cannot open a document with size equal to 0.

To create such a file, right-click on the desktop, point to New, and click Microsoft Office Word Document. The Packaging API cannot open a file that has a size equal to 0.

6.9 Some documents that cannot be opened by PPT pass the validation of the API

There are deviation between the ISO/ECMA specification and Office client app behaviors.  The API implementation is based on ISO/ECMA OOXML specification. So some documents pass the validation based on ISO/ECMA rules, but cannot be opened by PPT. This is designed behavior.
